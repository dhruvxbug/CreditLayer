────────────────────────────────────────────────────────────────
PROJECT OVERVIEW
────────────────────────────────────────────────────────────────

Problem:
DeFi lending requires 150%+ collateral. This excludes users with
real creditworthiness but limited capital. No protocol today uses
verifiable on-chain behaviour to offer lower collateral ratios
with privacy guarantees.

Solution — CreditLayer:
A lending protocol where borrowers earn a CreditScore NFT generated
by an AI agent that reads their DeFi history cross-chain, then
proves that score via a ZK proof to unlock loans at ratios as low
as 110% collateral — compared to Aave's 130–150%.

Core value propositions:
1. Privacy-first: borrowers never reveal raw wallet data to lenders
2. Composable: score is a SoulBound NFT usable across any protocol
3. AI-native: scoring logic is an autonomous agent, not a hard-coded formula
4. Capital efficient: lower collateral = more borrowing power for trusted users

Agent error handling:
- If HyperSync fails: fallback to subgraph, log warning, do not crash
- If Claude API fails: use deterministic weighted-sum score only, flag report
  as "AI_UNAVAILABLE" in response
- If wallet has zero DeFi history: return score of 100 with report
  "Insufficient on-chain history to generate a reliable credit score."
- Always return HTTP 200 with an error field rather than 4xx/5xx for
  scoring failures — the frontend should always get a structured response

SECURITY REQUIREMENTS
────────────────────────────────────────────────────────────────

- ReentrancyGuard on all state-changing lending functions
- Checks-Effects-Interactions pattern strictly enforced
- Score NFT is soulbound — cannot be traded, rented, or delegated
- ZK proof tied to msg.sender — proof from wallet A cannot be used by wallet B
  (include msg.sender as a public input in the Noir circuit)
- Oracle signer is a separate hot wallet — document key rotation procedure
- Slippage guard on collateral valuation: use a 5-minute TWAP on Chainlink
  rather than spot price to prevent flash-loan manipulation of liquidations
- All dollar amounts in 6-decimal USDC — no native token arithmetic without
  explicit unit documentation
